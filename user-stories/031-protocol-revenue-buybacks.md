# Protocol Revenue Buybacks

As a user, I want to see a portion of protocol revenue used for buybacks, so I
feel confident that token value is supported.

## Acceptance Criteria

- [ ] A configured share of protocol revenue is routed to a buyback wallet on a
      published cadence.
- [ ] The buyback wallet, allocation percentage, and execution cadence are
      visible in the UI.
- [ ] Each executed buyback is reported with timestamp, amount of revenue
      consumed, and tokens acquired, with on-chain transaction links.
- [ ] Historical buyback activity is queryable so users can audit the program
      over time.

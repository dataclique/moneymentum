# Keep The App Deployed And Reachable

As a user, I want to be able to use the platform so that Moneymentum is not just
a local development tool.

## Acceptance Criteria

- [ ] The repository contains deployment configuration for the frontend.
- [ ] The repository contains deployment configuration for the backend required
      by the frontend.
- [ ] Frontend configuration points at the deployed backend through checked-in
      configuration, not manual local edits.
- [ ] The backend exposes a smoke-testable health endpoint with a concrete
      contract: unauthenticated `GET /health` returns `200 OK` with
      `Content-Type: application/json` and body
      `{"status":"ok","version":"<semver-or-build-id>"}` on success, and a
      non-200 status on failure. Responses must set `Cache-Control: no-store` so
      intermediaries do not mask outages. CI smoke tests succeed only when the
      response status is 200, `status === "ok"`, and `version` is a non-empty
      string. An optional `GET /ready` follows the same contract for readiness
      checks.
- [ ] CI or deployment checks fail visibly when the deploy configuration is
      invalid.
- [ ] Automated post-deploy smoke verification: a CI job named
      `post-deploy-smoke-test` runs after deployment, issuing HTTP GETs to the
      deployed frontend URL and the backend `/health` endpoint from the public
      network. The job retries each endpoint up to 5 times with exponential
      backoff (2s, 4s, 8s, 16s, 32s). If retries are exhausted, the pipeline
      fails and the job triggers an automatic rollback to the previous known-
      good deployment, then notifies on-call via PagerDuty and the
      `#deploy-alerts` Slack channel and opens a follow-up ticket. If the
      project chooses not to auto-rollback, the job must instead halt the
      pipeline at a manual review gate that blocks promotion until a human
      acknowledges the failure.
- [ ] The README documents the master-branch deployment flow, including the
      post-deploy verification job and how to interpret its failures.

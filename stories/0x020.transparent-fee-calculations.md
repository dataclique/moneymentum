# Transparent Fee Calculations

As a user, I want transparent fee calculations so I know exactly how much is
allocated to buybacks or bounties.

## Acceptance Criteria

- [ ] Every fee charged by the protocol exposes its breakdown into named
      components (for example, buybacks, bounties, treasury).
- [ ] The breakdown is shown to the user before they confirm any action that
      incurs the fee.
- [ ] Historical fee allocations are queryable per component, with totals that
      reconcile to on-chain transfers.
- [ ] The formula and current parameter values used to compute each component
      are documented in the UI and reference the on-chain source of truth.
